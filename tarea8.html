¿Qué es HTTP?

HTTP (HyperText Transfer Protocol) es un protocolo de comunicación que permite la transferencia de información entre clientes y servidores a través de Internet. Es la base de la comunicación en la Web. Utiliza el protocolo TCP, generalmente a través del puerto 80, y está diseñado bajo el modelo cliente-servidor.

Cliente y Servidor en HTTP

Cliente HTTP

El cliente realiza peticiones al servidor. Generalmente es un navegador web, una app o una herramienta como curl o Postman. Solicita contenido como páginas, imágenes o envía formularios.

Servidor HTTP

El servidor recibe las solicitudes del cliente y responde con contenido o datos. Algunos ejemplos de servidores son Apache, Nginx o Microsoft IIS. Estos gestionan archivos web, bases de datos y aplicaciones.

Funcionamiento de HTTP

HTTP trabaja mediante un sistema de solicitud y respuesta. El cliente envía una solicitud al servidor, y este devuelve una respuesta que incluye un código de estado, encabezados y el contenido solicitado.

Ejemplo de solicitud:

GET / HTTP/1.1
Host: www.ejemplo.com

Ejemplo de respuesta:

HTTP/1.1 200 OK
Content-Type: text/html
[Contenido HTML]

Métodos HTTP

  • GET: Solicita datos. Usado para cargar páginas web.
  • POST: Envía datos al servidor. Ejemplo: enviar formularios.
  • PUT: Reemplaza completamente un recurso.
  • PATCH: Modifica parcialmente un recurso.
  • DELETE: Elimina un recurso.
  • HEAD: Solicita solo encabezados, sin el cuerpo.
  • OPTIONS: Informa qué métodos están permitidos por el servidor.

Códigos de Estado HTTP

1xx – Informativos

  • 100 Continue: El cliente puede continuar con su solicitud.

2xx – Éxito

  • 200 OK: Solicitud exitosa.
  • 201 Created: Recurso creado correctamente.

3xx – Redirecciones

  • 301 Moved Permanently: El recurso fue movido de forma permanente.
  • 302 Found: Redirección temporal.

4xx – Errores del cliente

  • 400 Bad Request: Error en la sintaxis de la solicitud.
  • 401 Unauthorized: Requiere autenticación.
  • 403 Forbidden: Acceso denegado.
  • 404 Not Found: El recurso no fue encontrado.
  • 429 Too Many Requests: Demasiadas solicitudes.

5xx – Errores del servidor

  • 500 Internal Server Error: Error general del servidor.
  • 502 Bad Gateway: El servidor recibió una respuesta inválida.
  • 503 Service Unavailable: El servidor no está disponible temporalmente.
